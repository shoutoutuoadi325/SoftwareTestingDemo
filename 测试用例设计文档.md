## 2. 数据库初始化记录（已完成）
### 2.1 导入动作
- 已在本机 MySQL 8.4 环境执行 `demo_db.sql` 导入到 `demo_db`。
- 导入方式: `mysql demo_db < demo_db.sql`（先执行 `CREATE DATABASE IF NOT EXISTS demo_db ...`）。

### 2.2 导入结果校验
导入后执行表和记录数校验，结果如下:

| 表名 | 记录数 |
| --- | --- |
| user | 7 |
| venue | 6 |
| news | 3 |
| message | 11 |
| order | 11 |

以上结果符合 `demo_db.sql` 基线数据，测试前置条件满足。

## 3. 用例设计说明
- 单元测试对象: `service` 层全部公开方法（含 `*VoService`）。
- 集成测试对象: `controller` 层主要接口（用户端 + 管理端）。
- 设计方法: 等价类、边界值、异常路径、状态流转断言。
- 关键状态定义:
  - 订单状态: 1 未审核, 2 已审核, 3 已完成, 4 失效/驳回
  - 留言状态: 1 未审核, 2 审核通过, 3 拒绝

## 6. 用例执行优先级建议
- P0（必须先执行）: 登录、下单、审核、留言审核、增删改关键路径。
- P1（第二批）: 分页、详情页、唯一性校验、VO 转换正确性。
- P2（补充）: 非法分页参数、时间格式错误等异常输入。
